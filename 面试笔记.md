## synchronized:

### synchronized（重量级锁）原理：

synchronized 给一个对象，该对象的mark word后两位表示为：10 ，其它位指向monitor 地址，monitor 中的owner指向获取锁的线程的锁记录的object reference，

后面再有其它线程则该线程进入monitor的entrylist队列中等待线程释放锁，释放锁后就把owner置为null，唤醒entrylist中阻塞等待的线程 

### synchronized（轻量级锁）原理：

 synchronized 给一个对象，此时在栈帧中创建了一个lock record 记录 ，该记录中有lock record 锁记录以及锁的对象地址object reference， 先让object reference 指向锁对象，并尝试使用cas替换锁对象的mark word ，将mark word 记录存入 锁记录中，如果cas成功，就将对象头中的mark word 就替换成了，锁记录地址和状态00，表示该对象已经加锁

锁升级：

当一个线程申请一个轻量级锁时，cas失败，就会经入锁升级流程，此时锁对象的mark word 就该为指向monitor地址，后两位也改为10，monitor的owner 指向持有锁的线程的object reference ，申请的线程进入entrylist队列

### wait 原理

锁后其它的进程无法执行，只能阻塞等待，直到线程,前提是线程必须拿到锁，之后才可以使用wait（）, 使用wait之后线程就放到monitor的waitset中，等待其它拿到锁的线程使用notifyAll（）// 激活所有在watiset中的线程 或者notify（）// 随机选择一个在waitset中的线程激活，此时会有虚假激活问题，需要用while循环解决

```java
        ReentrantLockTes reentrantLockTes2 = new ReentrantLockTes(c,d);

class ReentrantLockTes implements Runnable{
    ChopStick left;
    ChopStick right;
    ReentrantLockTes(ChopStick left, ChopStick right){
        this.left = left;
        this.right = right;
    }

    @Override
    public void run() {
       while (true){
           if (left.tryLock()){
               try {
                   if (right.tryLock()){
                       try {
                           System.out.println(Thread.currentThread().getName()+" 开始吃了");
                           Thread.sleep(1000);
                       } catch (InterruptedException e) {
                           throw new RuntimeException(e);
                       } finally {
                           right.unlock();
                       }

                   }
               }finally {
                   left.unlock();
               }
           }
       }
    }
}
class ChopStick extends ReentrantLock {
    String name;
    ChopStick(String name){
        this.name = name;
    }
}
```

### 1.可见性问题：

```java
public class ReentrantLockStu {
    static boolean run = true;
    public static void main(String[] args) {
        Thread thread = new Thread(new Runnable() {
            @Override
            public void run() {
                for (int i = 0; i < 10000000; i++) {
                  if (run){
                        System.out.println("run");
                    }else{
                        System.out.println("stop");
                        break;
                    }
                }
            }
        });
        thread.start();
        run = false;
    }
}
```

​	JIT自动优化了执行代码，因为子线程要频繁重复地从主存中读取run值，JIT会将run值缓存至自己工作内存的高速缓存中，减少对主存的值的访问，提高效率，但如果之后主线程修改了run的值，那么子线程就无法发现run的值改变依旧运行

解决方案：使用volatile关键字加给静态变量，或者使用synchronized 解决（synchronized工作时会将主存的内容读取到工作缓存中）

### 2 . 有序性问题（指令重排）：

多线程中，指令会重排，造成执行顺序的破坏，可以通过禁用指令重排解决改问题

```
class ReadAndWriteLockImp {
    private  HashMap<String, String> map  = new HashMap<>();
    ReentrantReadWriteLock lock = new ReentrantReadWriteLock();

    void write(String key,String value) throws InterruptedException {
        lock.writeLock().lock();

        try {
            System.out.println(Thread.currentThread().getName() + " 开始写");
            map.put(key, value);
            Thread.sleep(1000);
            System.out.println(Thread.currentThread().getName()+"写完了");
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        } finally {
            lock.writeLock().unlock();
        }
    }
    String read(String key) throws InterruptedException {
        lock.readLock().lock();

        try {
            System.out.println(Thread.currentThread().getName()+"开始读");
            Thread.sleep(1000);
            System.out.println(Thread.currentThread().getName()+"读完了,读到了"+map.get(key));
            return map.get(key);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        } finally {
            lock.readLock().unlock();
        }

    }
}
```

```java
public class FileTes {
    public static void main(String[] args) throws IOException {
        //
        searchStrNum("C:\\Users\\goodf\\Desktop\\leetcode\\LootCode\\src\\main\\java\\FileStreamReview\\filestu.txt",
                "C:\\Users\\goodf\\Desktop\\leetcode\\LootCode\\src\\main\\java\\FileStreamReview\\filestucopy.txt");
    }
    public static  void searchStrNum(String filePath,String str) throws IOException {
		//字节输入流
        FileInputStream fileReader = new FileInputStream(new File(filePath));
        //字节输入缓冲
        BufferedInputStream bufferIn = new BufferedInputStream(fileReader);
        //字节输出流
        FileOutputStream fileWriter = new FileOutputStream(new File(str));
        //字节输出缓冲
        BufferedOutputStream bufferOut = new BufferedOutputStream(fileWriter);
		
        byte[] bytes = new byte[1024];
        int length;
        while ((length = bufferIn.read(bytes))!= -1){
            System.out.println(new String(bytes,0,length));
            bufferOut.write(bytes,0,length);
        }
	
        System.out.println("完成");
        //关闭包装流原始流就会自己关闭
        bufferOut.close();
        bufferIn.close();
    }
}
```
